## Proposal

New API route for all txns:
https://api-mainnet.helius-rpc.com/v0/addresses/${address}/transactions?api-key=${apikey}
Returns: same format as above.

### Trade-off
Helius lacks historical token prices at txn time. Estimate via CoinGecko range API:
https://api.coingecko.com/api/v3/coins/${tokenCGID}/market_chart/range?vs_currency=usd&from=${from}&to=${to}

Caveat: 5-min snapshot intervals → price ±5min max outdated

## Implementation

### Database

4 new tables:
- `token_market_chart_5_m`: token snapshots for txns (5-min granularity)
- `transactions`: processed txn data from Helius raw JSON
- `transactions_meta`: coverage tracking per token + fetch timestamps
- `transactions_instructions`: decoded instruction data (self-referential)

### Functions

Transaction data:
- getter, fetcher, cacher, retriever, processor

Price data:
- getter, fetcher, cacher, retriever

Instruction handling:
- decoder, formatter

### Compatibility

Two approaches:
- **Backward compatible**: Moralis → transaction service, add formatter to match old schema
- **Rework routes + components**: Adapt all routes/UI for new data structure. Higher fidelity but heavy lift
